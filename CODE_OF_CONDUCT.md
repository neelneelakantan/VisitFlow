
# Contributor Code of Conduct

VisitFlow is a human‑centered project built around clarity, intentionality, and emotional safety.  
All contributors are expected to uphold these values when interacting with the project and its community.

This Code of Conduct outlines the expectations for behavior and the principles that guide collaboration.

---

# **1. Our Values**

VisitFlow is grounded in:

- **Respect** — Treat others with kindness and professionalism.  
- **Clarity** — Communicate clearly and constructively.  
- **Intentionality** — Make thoughtful, purposeful contributions.  
- **Humility** — Assume good intent and be open to feedback.  
- **Inclusivity** — Welcome contributors of all backgrounds and experience levels.  

These values reflect the spirit of the project itself.

---

# **2. Expected Behavior**

Contributors should:

- Be respectful and considerate in all interactions.  
- Provide constructive feedback, not criticism without context.  
- Collaborate with empathy and patience.  
- Keep discussions focused on the work, not the person.  
- Follow the project’s architecture and design principles.  
- Ask questions when unsure — clarity is valued over assumptions.  

---

# **3. Unacceptable Behavior**

The following behaviors are not acceptable:

- Harassment, discrimination, or personal attacks  
- Dismissive or hostile communication  
- Introducing features that violate the project philosophy  
- Ignoring architectural boundaries or intentionally creating drift  
- Sharing private or sensitive information  
- Any behavior that makes others feel unsafe or unwelcome  

---

# **4. Scope**

This Code of Conduct applies to:

- GitHub issues and discussions  
- Pull requests and code reviews  
- Project documentation  
- Any public or private communication related to the project  

---

# **5. Reporting Issues**

If you experience or witness behavior that violates this Code of Conduct:

- Open an issue marked **“Code of Conduct”**, or  
- Contact the project maintainer privately (email optional and not required)  

Reports will be handled respectfully and confidentially.

---

# **6. Enforcement**

The project maintainer may take appropriate action, including:

- Clarifying expectations  
- Requesting changes to behavior  
- Closing or rejecting pull requests  
- Temporarily or permanently banning contributors  

Enforcement decisions will prioritize the health and safety of the community.

---

# **7. Acknowledgment**

By participating in VisitFlow, you agree to uphold this Code of Conduct and contribute to a respectful, intentional, and supportive environment.
